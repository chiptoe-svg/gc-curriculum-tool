# Top-Level Wiki index.md OKF Conformance — Design

**Date:** 2026-06-14
**Status:** approved design (operator brainstorm 2026-06-14), pre-plan
**Origin:** Deferred item A of the two remaining OKF follow-ons after Increment #2 (wiki frontmatter alignment). Increment #2 brought the 4 narrative page types + per-section `index.md` hubs to OKF v0.1 but deliberately left the **top-level root `index.md`** out of scope. This increment closes that gap so the *whole* wiki is OKF-conformant.

## Decisions made in the brainstorm (2026-06-14)

1. **Stamp the root index AND enforce it in the lint.** Extend `gc-wiki-lint` to run the `okf-frontmatter-missing` check on the root `index.md` (skipping the type-specific checks, like the section indexes). Conformance is a guaranteed, regression-proof property of the whole wiki including the root.
2. **`title`/`description` are LLM-authored** (update the prompt's index block, left untouched in Increment #2); `slug`/`tags`/`timestamp`/`resource` are machine-stamped — identical to every other page. [defaulted, operator-approved]
3. **`resource` for the root = `<base>/wiki`** (the wiki home), not `<base>/wiki/index/index`. [defaulted, approved]
4. **`tags: [index]`**; preserve the domain stats keys `total_snapshots`/`total_courses_with_snapshots`/`input_hash`. [defaulted, approved]
5. **Out of scope:** broken-wikilink checking on the root index (its links are LLM-generated from the live page set; separate concern) — `okf-frontmatter-missing` only.

## Context (grounding findings)

- The root `index.md` is the LLM-generated dashboard: H1 "GC Curriculum Knowledge Base" + a narrative + course/competency/target/concept tables (each row a `[[wikilink]]` + last-snapshot date + summary). Current frontmatter: `type: index`, `updated_at`, `total_snapshots`, `total_courses_with_snapshots`, `input_hash`. No OKF keys.
- It is generated by `wiki-update` (in the page list at `update.ts:149` as `{ type:'index', slug:'index', path:'index.md' }`, substrate at `:900`) and is currently **excluded** from the OKF post-stamp by the `if (p.path !== 'index.md')` guard at `update.ts:1067`.
- It is **not linted** — `lint.ts` only scans the 4 `WIKI_PAGE_TYPES` dirs; the root `index.md` lives at the repo root.
- `lib/ai/wiki/okf-frontmatter.ts` (Increment #2) already provides `stampOkfFrontmatter`, `okfResource`, `deriveTags`, `setFrontmatterLine`, `readFrontmatterScalar`, `OKF_REQUIRED_KEYS`. `deriveTags('index', …)` returns `['index']` (default case). `okfResource('index', 'index')` currently returns `<base>/wiki/index/index` (TYPE_TO_DIR has no `index` entry → falls through) — this is the one thing to fix.

## Target frontmatter (root `index.md`)
```yaml
---
type: index
title: "GC Curriculum Knowledge Base"      # LLM-authored (regen) / H1-derived (backfill)
description: "<one-line wiki summary>"      # LLM-authored / first-body-sentence (backfill)
slug: index                                # machine-stamped
tags: [index]                              # machine-stamped
timestamp: <ISO>                           # machine-stamped (was updated_at)
resource: <BASE>/wiki                      # machine-stamped (root home, special-cased)
total_snapshots: 7                         # preserved domain key
total_courses_with_snapshots: 5            # preserved domain key
input_hash: bf940566b9d3                   # preserved
---
```
`<BASE>` = `process.env.WIKI_PUBLIC_ORIGIN ?? 'http://gcworkflow.clemson.edu:3000'` (same as the rest).

## Components

1. **`lib/ai/wiki/okf-frontmatter.ts`** — special-case `okfResource`: when `type === 'index'`, return `${base}/wiki` (slug ignored). Only the root index reaches `okfResource` with `type:'index'` (the four section indexes build their own `resource` inside `buildSectionIndex`, which is unchanged). No other change to the pure module.

2. **`lib/ai/wiki/update.ts`** — remove the `if (p.path !== 'index.md')` guard around the OKF stamp (lines ~1064-1068) so the root index is stamped on every regen. The existing slug derivation (`p.path.replace(/^.*\//,'').replace(/\.md$/,'')`) already yields `'index'` for `index.md`. `title`/`description` come from the LLM output; the stamp supplies `slug`/`tags`/`timestamp`/`resource`. Update the comment that currently says the index is "out of OKF scope."

3. **`lib/ai/prompts/wiki-update.md`** — update the index example block (the one left unchanged in Increment #2) to author `title` + a one-sentence `description` and use `timestamp:` (was `updated_at:`), preserving `total_snapshots`/`total_courses_with_snapshots`. The "do not author machine fields" rule already added in Increment #2 covers `tags`/`timestamp`/`resource`/`slug` for the index too.

4. **`scripts/wiki-backfill-okf.ts`** — extend to migrate the root `index.md`:
   - New exported pure `deriveTitle(content)`: the first `# ` heading text (wikilink-stripped, `"`→`'`); fallback `'GC Curriculum Knowledge Base'`.
   - A `backfillRootIndex(content)`: `stampOkfFrontmatter(content, { slug: 'index' })`, then add `title` (via `deriveTitle`) if absent and `description` (via the existing `deriveDescription`, with title→`'index'` fallback) if absent. Idempotent.
   - In `main()`, after the type-dir loop and before/after `rebuildSectionIndexes`, read `<root>/index.md`, apply `backfillRootIndex`, write if changed (counted in the migrated total).

5. **`lib/ai/wiki/lint.ts`** — after the existing type-dir page loop, read the root `index.md` (if present) and push an `okf-frontmatter-missing` error listing any of `OKF_REQUIRED_KEYS` absent from its frontmatter (page label `index.md`). No orphan/missing-section/broken-wikilink for it. `allSlugs` and the existing checks for the 4 type dirs are unchanged.

## Data flow
Regen: LLM authors the root index prose + `title`/`description` → OKF post-stamp (now incl. `index.md`) adds `slug`/`tags`/`timestamp`/`resource` + `input_hash` → written by git-ops → `gc-wiki-lint` gates (root index OKF-checked). Backfill: one-time `backfillRootIndex` over the existing root file using the shared pure helpers + H1/body-derived title/description.

## Testing
- **`okfResource` (pure):** `okfResource('index', 'index', 'http://x')` → `http://x/wiki`; the four narrative types unchanged (e.g. `okfResource('course', 'gc-1010', 'http://x')` → `http://x/wiki/courses/gc-1010`).
- **`deriveTitle` (pure):** first `# ` heading; wikilink-stripped; fallback when no heading.
- **`backfillRootIndex` (pure):** legacy root index → OKF keys present (title from H1, description, slug `index`, tags `[index]`, timestamp from `updated_at`, resource `<base>/wiki`); `total_*`/`input_hash` preserved; `updated_at`/`type:index` correct; idempotent (re-run identical).
- **Lint:** root `index.md` missing an OKF key → one `okf-frontmatter-missing` error (page `index.md`); clean when all present; the 4-type-dir checks unaffected (regression).
- **Integration:** `pnpm wiki:backfill-okf` migrates the root index; `pnpm wiki:lint` → 0 `okf-frontmatter-missing` (root included; the ~45 pre-existing broken-wikilink errors are unchanged). Commit/push the wiki repo; update STATE.md.

## Out of scope (deferred)

- `/wiki/graph` view.
- broken-wikilink enforcement on the root index.
- Any change to the root index's body/tables or its `total_*` stat semantics.
